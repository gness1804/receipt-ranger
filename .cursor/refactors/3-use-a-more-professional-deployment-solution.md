# Use a more professional deployment solution.

## Working directory

`~/conductor/workspaces/receipt-ranger/`

## Contents

Streamlit Community Cloud (SCC) is great for amateurs playing with applications, but if I want my application to actually seem professional, I need to use something more appropriate. For instance, SCC always needs for the application to be woken up, which isn't true for a real production app. There's also GitHub stuff and other things that make it look amateurish, so I need a professional deployment solution that works well for SCC. 

A major pain point in the past has been deploying with Streamlit, however. I do not want to change the whole application to use a different technology than Streamlit. I want to keep Streamlit but not use the Streamlit Community Cloud.  We need to find the right solution, maybe on AWS, that works with Streamlit and this application. Before doing any actual work, we will need to invoke a sub-agent to research different options and to do this rigorously with full knowledge of how the application works.
